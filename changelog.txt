MCM Security Hardener — Changelog

= 1.15.1 (2026-06-02) =
* UX: Login-URL mail is nu een nette HTML-mail met klikbare knop
  "Naar inlogpagina" in plaats van een plain-text bericht
* Persoonlijke aanhef per ontvanger ("Hallo Marco," in plaats van
  "Beste,") op basis van first_name → display_name → generiek
* Vriendelijkere intro en duidelijker visueel onderscheid tussen URL,
  uitleg en metadata
* Waarschuwingsbox als er geen aangepaste inlog-slug is ingesteld
* Plain-text fallback via PHPMailer AltBody voor mail-clients die
  geen HTML renderen
* Geen functionele wijzigingen — verzendlogica en ontvangers ongewijzigd

= 1.15.0 (2026-06-02) =
* Profielen (Basic/Standard/Strict/Staging) sturen nu ook de nieuwere
  toggles uit v1.12-v1.14
* Basic + Standard + Strict + Staging: registration_honeypot AAN
* Basic + Standard + Strict: block_disposable_email AAN
  (Staging UIT — test-emails moeten kunnen)
* Basic + Standard + Strict + Staging: skip_admin_email_confirmation AAN
* Strict: block_non_admin_backend AAN (klant-rollen niet in wp-admin)
* Belangrijk: ook bugfix — voorheen werden v1.12-v1.14 toggles bij elke
  profile-apply op false gezet omdat ze niet in all_toggle_keys() stonden

= 1.14.0 (2026-06-02) =
* Nieuw: Backend toegang sectie (MCM_Backend_Access class)
* Skip admin email confirmation — schakelt het halfjaarlijkse
  "controleer admin email"-tussenscherm uit voor administrators
  (default AAN — universeel irritant)
* Block non-admin backend access — redirect ingelogde users zonder
  toegestane rol weg van /wp-admin/ (default UIT, bewuste keuze per site)
* Whitelist van rollen instelbaar via textarea (default: administrator + editor)
* Redirect-URL instelbaar (default homepage)
* AJAX/REST/WP-CLI/cron altijd doorgelaten
* MCM-eigenaars worden nooit geblokkeerd
* Filters: mcm_allowed_backend_roles, mcm_unauthorized_backend_redirect

= 1.13.0 (2026-06-01) =
* Fix: Human Verification draaide niet op WooCommerce my-account login/register/lostpassword
  Het vinkje werd niet getoond op WC-formulieren, maar de validatie liep wel — gevolg:
  "menselijke verificatie nog niet voltooid" zonder dat klanten ooit het vinkje gezien hadden.
* Hooks toegevoegd: woocommerce_login_form, woocommerce_register_form, woocommerce_lostpassword_form
* render_field() roept enqueue_styles() aan zodat CSS ook geprint wordt op pagina's waar
  login_enqueue_scripts niet draait
* enqueue_styles() is nu idempotent (static $printed) — geen dubbel printen op wp-login.php

= 1.12.0 (2026-05-28) =
* Nieuw: Registratiebescherming (MCM_Registration_Protection class)
* Honeypot-veld bij registratie — gevuld door bots, onzichtbaar voor mensen
* Wegwerpdomein-filter — weigert tijdelijke emailadressen (mailinator, yopmail, 10minutemail, etc.)
* Werkt op zowel WP-standaard als WooCommerce registratie
* Eigen wegwerpdomeinen toe te voegen via textarea in instellingen
* Filter 'mcm_blocked_email_domains' voor programmatische uitbreiding
* Werkt complementair naast Human Verification (timing vs honeypot)

= 1.11.0 (2026-05-28) =
* Nieuw: WordPress major-update compatibiliteits-check (MCM_Update_Compat_Check)
* Toont op Dashboard → Updates een bewustwordings-blok bij major WP-update
* Per actieve plugin vergelijking 'Tested up to' uit readme.txt met de aankomende WP-versie
* Status: Compatibel / Niet getest / Onbekend
* Zwijgt bij minor-updates (6.7.1 → 6.7.2)
* Alleen zichtbaar voor MCM-eigenaars
* Verbetering: is_mcm_owner() herkent owners nu ook via emailadres (= MCM_Notifier::notify_email())
  zodat MCM_SECURITY_OWNERS-constante niet per se nodig is

= 1.10.0 (2026-05-26) =
* Nieuw: User Audit sectie (MCM_User_Audit class)
* Toont alle users met rol Administrator/Editor/Author/Contributor
* 1-klik downgrade naar MCM Klant (mits Site Optimizer aanwezig) of Subscriber
* Skipt automatisch MCM-eigenaars en super-admins (multisite)
* Admin-downgrade alleen toegestaan voor users met manage_options
* Nonce-beveiligde admin-post URL flow

= 1.9.0 (2026-05-26) =
* Nieuw: WP_DEBUG productie-watchdog (MCM_Debug_Watchdog class)
* Detecteert WP_DEBUG=true op productie-omgevingen via wp_get_environment_type()
* Admin-notice (oranje voor WP_DEBUG, rood voor WP_DEBUG_DISPLAY = info-leak)
* 1×/24u mail naar de eigenaar zolang situatie aanhoudt — geen mail-moeheid
* Zwijgt op staging/dev-omgevingen
* Uit te schakelen via filter of constante MCM_SECURITY_DISABLE_DEBUG_WATCHDOG
